## GitHub Pages SPA 路由修复（2026-05-27）

- 问题1：访问 /dashboard/ 后跳转到 /emergency/... 导致 /dashboard/ 前缀丢失
  → 原因：BrowserRouter 缺少 basename，React Router 不知道自己运行在 /dashboard/ 下
  → 修复：`<BrowserRouter basename={import.meta.env.PROD ? '/dashboard/' : '/'}>`
- 问题2：直接访问子路径（如 /dashboard/emergency/station-chief-v2）报 404
  → 修复：添加 public/404.html + index.html 还原脚本（将原始路径存 sessionStorage，跳到 /dashboard/，应用加载时恢复带 /dashboard/ 前缀的完整路径）
- 注意：只有带 /dashboard/ 前缀的 URL 才能正常工作，/emergency/... 这种不带前缀的是用户站点根域名下，GitHub 不会路由到项目站点
