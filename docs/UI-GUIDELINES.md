# UI/UX 设计规范

## 筛选器设计原则

### 1. 重置按钮
**原则**：当筛选条件 ≥ 3 个时，必须提供「重置」按钮。

**原因**：
- 用户可能不小心选择了多个筛选条件，导致结果为空
- 提供一键清除比逐个取消更高效
- 符合「容错设计」原则

**实现示例**：
```tsx
<div style={{ display: 'flex', gap: 16, alignItems: 'flex-start', flexWrap: 'wrap' }}>
  {/* 筛选条件们 */}
  <select>...</select>
  <select>...</select>
  
  {/* 重置按钮 - 放在右侧或底部 */}
  <button onClick={() => {
    setFilter1('all')
    setFilter2('all')
    setKeyword('')
  }}>
    重置筛选
  </button>
</div>
```

**视觉建议**：
- 使用次要按钮样式（边框 + 浅色背景）
- 颜色与主色调区分开（如黄色/橙色系表示"警告/重置"）
- 位置：通常在所有筛选条件右侧，或底部居中

### 2. 筛选状态提示
**原则**：当某个筛选条件处于激活状态时，应提供视觉提示。

**实现**：
```tsx
<span>状态 {hasActiveFilter && <span style={{ color: '#4F46E5' }}>（已筛选）</span>}</span>
```

### 3. 分页设计
**原则**：当数据量 > 50 条时，应提供分页。

**默认配置**：
- 每页 50 条
- 筛选变化时自动回到第 1 页
- 显示：共 X 条，当前第 Y / Z 页
- 控件：首页、上一页、页码（最多显示5个）、下一页、末页

### 4. 加载状态
**原则**：异步加载数据时，必须显示加载状态，避免用户困惑。

```tsx
if (loading) {
  return <div>加载中...</div>
}
```

### 5. 空状态
**原则**：筛选结果为空时，应提供友好的提示信息。

```tsx
{sortedData.length === 0 ? (
  <div>未找到匹配结果</div>
) : (
  // 正常渲染列表
)}
```
